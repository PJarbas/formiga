## Codebase Patterns

- **Build script:** `npm run build` compiles TypeScript, copies server HTMLs, injects version, and copies www/ → dist/www/. The build script must `rm -rf dist/www` before `cp -r` to avoid nesting (dist/www/www/).
- **Tests:** Use `node:test` + `node:assert/strict`. Test files in `tests/` and `src/**/*.test.ts`. Excluded from `tsconfig.json` compilation — run directly with `node --test`. Must use Node.js v22 binary explicitly on this system (system default is v18).
- **HTML:** `www/index.html` is the website root. CSS and JS files are linked separately (styles.css, scripts.js). Screenshots go in `www/assets/`.
- **Commit format:** All commits must end with `Co-Authored-By: Tamandua <tamandua@tetradactyla.org>`.
- **CSS testing:** CSS tests run against dist/www/styles.css (after build). Regex-based tests must account for custom properties (variables) rather than expecting inline hex values in body rules. Print media queries have nested braces, so `[^}]*` won't work across the full block — test for `@media print` presence and `display:none` separately.
- **CSS hamburger menu:** Mobile hamburger menu uses CSS checkbox hack (`<input type="checkbox" id="nav-toggle">` with `:checked` pseudo-class). No JavaScript needed for mobile nav toggle.
- **CSS pseudo-element decoration:** Decorative background patterns and accents should use `::before`/`::after` pseudo-elements with `pointer-events: none` so they don't interfere with clicks/interactions.
- **CSS counter for numbered lists:** Use `counter-reset` on the container and `counter-increment` + `::before` with `content: counter(name)` on list items. Combine with `border-radius: 50%` for circular number badges. Alternating colors use `:nth-child(n)` with different `border-left-color` and `::before` color pairs.
- **JS progressive enhancement:** All JS is vanilla, wrapped in `DOMContentLoaded` with `'use strict'`. Feature detection before API use (e.g., `'IntersectionObserver' in window`). JS file must import no external libs — use `require`/`import` only for Node APIs (tests use them for file reading).
- **JS tests:** Test files for scripts.js verify structural contracts (class names, API references, event listeners) since full behavioral testing requires a browser. Tests run against dist/ after build. Module-scope file reading, no `before` hooks.
