## Codebase Patterns
- All autoresearch logic lives in `src/autoresearch/autoresearch.ts`
- Tests import from `../../dist/autoresearch/autoresearch.js` (compiled output)
- Tests use `node:test` and `node:assert/strict`
- Git helpers via `spawnSync` in tests; a `git()` test helper wraps `spawnSync("git", ...)`
- Temp dirs via `fs.mkdtempSync(path.join(os.tmpdir(), ...))`
- `revertExperimentChanges` uses `git status --porcelain` to split tracked/untracked files
- Build: `npm run build` (tsc + copy HTML + inject version)
- Test: `npm test` (node --test tests/*.test.ts src/**/*.test.ts)
- The pre-existing prompt-mode loop test hangs because it tries to invoke `pi` which isn't available
- `commitAutoresearchResult` uses two-step: `git add -A` then `git reset -- <protectedPaths>` to unstage autoresearch files before commit
- When only autoresearch files are changed, git commit fails with "nothing to commit" — handled gracefully by returning `undefined`
- `logExperiment`'s `revertDiscard` option only reverts for `status === "discard"` — crash/checks_failed must be reverted separately by the caller
- `runPiAgent` spawns `pi` as a bare binary name (relies on PATH), not a resolved path
- `parseAgentFields` parses STATUS/CHANGES/HYPOTHESIS/LEARNED/NEXT_FOCUS from agent output text
- `runLoopIteration` runs experiment via session config command, accepts optional `command` override
- `parseDuration` in cli.ts handles `<number><s|m|h|d>` format
- CLI action handlers follow pattern: parse options via `readOption`, call function, output result
- `get<Thing>Help()` convention for per-command help functions
- `--help` dispatch in `main()` runs before any command execution
- `getUsageText()` lists all top-level commands
- CLI tests use isolated HOME and TAMANDUA_STATE_DIR, spawn CLI via `spawnSync` with `process.execPath`
- test isolation guard prohibits `...process.env` in test files
