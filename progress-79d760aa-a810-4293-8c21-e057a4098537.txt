## Codebase Patterns
- All workflows are in `workflows/<id>/` with `workflow.yml` + `agents/<agent-id>/{AGENTS,SOUL,IDENTITY}.md`
- Shared agent personas live in `agents/shared/<persona>/{AGENTS,SOUL,IDENTITY}.md` and are referenced via `../../agents/shared/<persona>/*.md`
- Workflow specs are parsed via `loadWorkflowSpec()` from `dist/installer/workflow-spec.js`
- `resolveBundledWorkflowsDir()` from `dist/installer/paths.js` gives the workflows root
- Tests use `node:test` with `node:assert/strict`
- The `tamandua-agents` skill is required on all workflow agents and is validated by tests
- Polling config uses `model: default` and `timeoutSeconds: 120` for these workflows
- Shared setup persona captures `ORIGINAL_BRANCH=$(git branch --show-current)` before creating a feature branch
- Merge workflows use `git merge --squash` + `git commit -F <tempfile>` (NOT `git commit -m` and NOT `gh pr create`)
- Setup and verifier agent directories are empty — they wire shared personas purely through workflow.yml workspace.files paths
- `workflow-validation.test.ts` dynamically iterates ALL workflow directories; adding a new workflow requires adding an explicit assertion for its inclusion in the "finds at least N bundled workflows" check
