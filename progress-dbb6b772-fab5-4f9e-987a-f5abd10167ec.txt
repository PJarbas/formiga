## Codebase Patterns
- CLI entry point is `src/cli/cli.ts` with `async function main()` at line 1126
- Every command has a `get<Thing>Help()` function returning a multi-line help string
- `--help` dispatch happens at the top of `main()` (line 1132) before any command execution
- `getUsageText()` at line 1065 lists all top-level commands for `tamandua --help`
- New command dispatch blocks in `main()` go BEFORE the `if args.length < 2` fallthrough at line 1698
- Tests use `node:test` and `node:assert`, with temporary HOME directories for isolation
- The test `cli()` helper spawns `bin/tamandua` wrapper with temp HOME/state dir
- Project requires Node.js >= 22 for `node:sqlite` built-in module
- Build: `npm run build` (TypeScript compilation + version injection)
- Test: `npm test` (runs `node --test tests/*.test.ts src/**/*.test.ts`)
- Service status imports come from `../server/daemonctl.js` (isRunning, getDaemonStatus, getMcpStatus, isMcpRunning, etc.)
- Path resolution via `../installer/paths.js` (resolveSourcePath, resolveSkillPath)
- Run listing via `../installer/status.js` (listRuns)
- **IMPORTANT**: Do NOT import from `dist/cli/cli.js` in tests — it calls `main()` which runs `process.exit(1)` on no-args. Place testable functions in separate modules (like `src/cli/status-format.ts`) and import those from tests.
- daemonctl status functions: getDaemonStatus returns `{running: bool, pid: number|null, port: number}`, getMcpStatus/getControlPlaneStatus return `{running: bool, pid: number|null, port: number, endpoint: string}`
- Default ports: MCP = 3338 with endpoint /mcp, Control-plane = 3339 with endpoint /control/health, Dashboard port from file
- `git rev-parse HEAD^{tree}` returns a 40-char hex SHA representing the source tree content — used for installation verification
- `readVersionStatus()` from `../lib/version-check.js` returns `VersionStatus { updateAvailable, currentHead, remoteHead, checkedAt }`
- `execSync` from `node:child_process` can be dependency-injected by passing a function matching `(cmd: string) => string` signature
- `listRuns(limit)` from `../installer/status.js` returns `RunInfo[]` — each has `id`, `workflowId`, `task`, `status`, `createdAt`, `updatedAt`, `stepSummary?`, `tokensSpent`
- `ps -eo pid,etime,args --no-headers` gives PID, elapsed time, and full command for process discovery
- `isRunning()` from daemonctl returns `{ running: true; pid: number } | { running: false }` — used to gate process discovery
