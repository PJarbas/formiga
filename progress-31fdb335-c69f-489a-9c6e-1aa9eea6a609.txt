## Codebase Patterns

- All TypeScript source lives in `src/`; compiled output goes to `dist/`.
- Tests import from `../../dist/cli/...` (not `../../src/`), so build must succeed before test.
- Tests use `node:test` and `node:assert/strict`.
- The project uses `tsc` with `tsconfig.json` for building.
- The `pi-stream-parser.ts` module provides `parsePiOutputStream()` for parsing pi JSONL output.
- `bin/tamandua` is a shell wrapper that invokes `dist/cli/cli.js`.
- `parsePiOutputStream` accepts `Iterable<string> | AsyncIterable<string>` — an array of strings works in tests.
- Fake ChildProcess for tests: EventEmitter with PassThrough stdout/stderr, emit "close" on setImmediate.
- `extractAssistantText()` from pi-stream-parser walks `message.content[]` array looking for `type: "text"` blocks.
- `asRecord()` helper in pi-stream-parser excludes arrays (nulls them out).
- Wizard modules follow pure-function pattern with dependency injection where I/O is needed (PiSpawnFn, readline injection).
- Shared validation uses a `validateArgv` helper with required prefix, allowed flag set, and checkPromptFlag boolean.
- Shell quoting: single-quote everything, escape embedded single quotes via `'\''`.
- `summarizeAutoresearch(cwd)` from `src/autoresearch/autoresearch.ts` returns `{ exists: boolean, ... }` — use to check if a session is initialized.
- Command execution in orchestrator uses `spawnAndWait` with `stdio: "inherit"` so user sees live output.
- CLI help subprocess tests use `spawnSync` with `cleanChildEnv` and isolated temp HOME/state dirs. Always clean up temp dirs in a `finally` block. Help text may wrap at terminal width — use `\s+` in regex to span line breaks.

## 2026-05-27T21:29:00Z - US-003: Create command argv validator and shell renderer
- Implemented `src/cli/wizard-commands.ts` exporting:
  - `validateInitArgv(argv)` — validates init argv against allowed init flags (--goal, --metric, --unit, --direction, --command, --metric-regex, --checks-command, --cwd, --overwrite). Rejects --prompt and unknown flags.
  - `validateLoopArgv(argv)` — validates loop argv starting with ["autoresearch", "loop", "--prompt"]. --prompt is a flag (no value allowed). Rejects unknown flags (only --target-metric, --max-iterations, --max-consecutive-failures, --timeout, --cwd allowed).
  - `shellQuote(arg)` — single-quote shell escaping with embedded single-quote handling.
  - `renderShellCommand(argv, binaryName?)` — renders an argv array as a pasteable shell command with the tamandua prefix.
  - `renderWizardCommands(result, binaryName?)` — validates argvs and renders display string (single loop command or `init ; loop`), throws on invalid argv.
- Shared `validateArgv()` helper with `checkPromptFlag` parameter handles both init and loop validation.
- Files: `src/cli/wizard-commands.ts`, `src/cli/wizard-commands.test.ts`
- 42 tests all pass.
- **Learnings:**
  - `validateLoopArgv` uses `checkPromptFlag=true` on prefix `["autoresearch", "loop"]` to enforce `--prompt` presence and no-value constraint.
  - The `RenderedWizardCommands` interface includes structured argv arrays (not just display string) so the orchestrator can execute them via spawn rather than shell strings.
