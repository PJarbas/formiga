## 2026-05-13 - US-005: Add regression tests for STORIES_JSON guard retry attribution and setup retry_feedback

### Code Changes
- **src/installer/step-ops.ts**: Enhanced completeStep STORIES_JSON guard to search for downstream loop-over-stories beyond the immediate next step when the completing step's input_template mentions STORIES_JSON. Previously the guard only checked `step_index + 1`, so workflows where an intermediate step (e.g., setup) sits between planner and loop (feature-dev-merge: plan → setup → implement) would blame the intermediate step instead of the story-producing planner. Now story-producing steps (whose input_template includes "STORIES_JSON") trigger a wider search (`step_index > ?`) for downstream loops. Non-story-producing steps still only check the immediate next step, preserving security-audit behavior. Added `input_template` to completeStep's SELECT query.

### Tests Added
In **tests/step-ops.test.ts** (new describe blocks):
1. `"blames planner (not setup) when planner omits STORIES_JSON and there is an intermediate step before the loop"` — Simulates e49c370d: plan(idx0)→setup(idx1)→implement(idx2,loop). Planner completes without STORIES_JSON. Asserts: result.status="retrying", result.detail includes STORIES_JSON, planner reset to pending (retry_count=1), setup stays waiting.
2. `"does not blame planner across intermediate steps when stories already exist"` — Re-planning scenario: planner retry with valid STORIES_JSON. Asserts planner marked done, setup advances to pending.
3. `"claimStep resolves setup input with retry_feedback when retry_count > 0"` — Setup step with retry_count=1 and output=prior failure. Asserts resolvedInput contains prior error text.
4. `"claimStep resolves setup input with empty retry_feedback when retry_count is 0"` — First-attempt setup. Asserts resolvedInput does not contain `[missing: retry_feedback]`.

In **tests/workflow-validation.test.ts**:
5. `"feature-dev family setup prompts include {{retry_feedback}} placeholder"` — Validates that feature-dev, feature-dev-merge, and feature-dev-github-pr workflow setup step inputs all contain `{{retry_feedback}}`, RETRY FEEDBACK section label, and Instructions section.
