## Codebase Patterns
- Workflow YAML files live at `workflows/<id>/workflow.yml` with id matching directory name
- Agent persona files follow AGENTS.md/IDENTITY.md/SOUL.md pattern in `workflows/<id>/agents/<agent-id>/`
- Every agent must have `tamandua-agents` skill declared in workspace.skills
- Agent workspace.files paths are relative to the workflow directory
- Tests in `tests/workflow-validation.test.ts` auto-discover all workflow directories and validate them
- Workflow YAML requires: id (string), agents (non-empty array), steps (array)
- Step requires: id (string), agent (string)
- Agent requires: id (string), workspace (object with baseDir)
- Valid agent roles are: analysis, coding, verification, testing, pr, scanning
- Build command: `npm run build` (tsc + HTML copy + version injection)
- Test command: `npm test`
