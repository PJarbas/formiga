## Codebase Patterns

- `src/db.ts` uses `DatabaseSync` from `node:sqlite` with a singleton pattern (`getDb()`)
- Migrations use `CREATE TABLE IF NOT EXISTS` + `ALTER TABLE ADD COLUMN` with PRAGMA table_info checks
- Indexes use `CREATE INDEX IF NOT EXISTS`
- Tests isolate DB via temp HOME directory and `process.env.HOME` override
- Tests import from `../dist/db.js` (compiled output)
- Source files use relative `.js` extension imports for Node16 module resolution
- `as unknown as Type` pattern needed for SQLite row casts due to `Record<string, SQLOutputValue>` mismatch
- CLI tests use `cleanChildEnv` from `tests/helpers/test-env.ts` which computes `TAMANDUA_DB_PATH` from `TAMANDUA_STATE_DIR` (not HOME directly — DB path is `$TAMANDUA_STATE_DIR/tamandua.db`)
- CLI tests spawn tamandua via `spawnSync("/bin/sh", [wrapperPath, ...args])` — the `cli()` helper creates isolated envs but can accept `env` overrides for sharing HOME/state across calls
- Dashboard tests use `createDashboardServer(0)` to get a random port, then `fetch()` against `http://127.0.0.1:${address.port}`
- Route registration order matters: `/api/autoresearch/sessions/:id` must be registered before `/api/autoresearch/sessions` to avoid prefix conflicts
- Route regex for URL-encoded IDs needs `%` in the character class: `[a-zA-Z0-9_\/.%~-]`
- `parseDuration(input: string)` parses `<number><unit>` where unit is d, h, or m (e.g. 7d, 24h, 30m) and returns milliseconds
- The `worktree prune` command provides a reference pattern: require a flag, parse duration, query DB, iterate, delete, report
- Dynamic import (`await import("../db.js")`) in CLI action handlers for lazy-loading DB functions
