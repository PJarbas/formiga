## Codebase Patterns
- All CLI commands live in src/cli/cli.ts (single file, ~881 lines)
- Workflow install/uninstall use the target variable pattern: `const isAll = target === "--all" || target === "all"`
- The top-level `tamandua install` (no workflow group) already iterates over `listBundledWorkflows()` and calls `installWorkflow()` for each
- Tests use `npm test` which invokes `node --test` — integration tests use isolated temp HOME dirs
- Build: `npm run build` runs `tsc -p tsconfig.json` + copies HTML + injects version
- Integration tests that exercise `workflow install` must seed `~/.pi/agent/settings.json` in the temp HOME because `installWorkflow()` calls `readPiConfig()` which throws on ENOENT
- The `listBundledWorkflows()` function resolves from `dist/installer/paths.js` → `../../workflows` — it ignores HOME and always reads from the checkout's bundled workflows, so the test environment doesn't affect which workflows are discoverable
