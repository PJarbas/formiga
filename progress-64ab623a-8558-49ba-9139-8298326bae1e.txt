## Codebase Patterns

- Tests import from `../../dist/...` — they run compiled JS, not TS directly
- Tests use `fs.mkdtempSync(path.join(os.tmpdir(), ...))` for temp HOME isolation
- `initGitRepo(dir)` helper in run.test.ts creates a minimal git repo for testing
- DB queries in tests use dynamic `await import("../../dist/db.js")` to get the isolated DB instance
- `seededContext` is a Record<string, string> built before DB insertion in runWorkflow()
- Worktree mode stores `worktree_origin_sha` in context automatically via `ManagedRunWorktree`
- `execFileSync` is used for git commands; stderr must be suppressed (`stdio: ["ignore", "pipe", "pipe"]`)
- Context values are always stored as strings (`String(noHurrySaveTokensMode ?? false)`)
- All boolean flags in parseWorkflowRunArgs default to `undefined` when not provided, and are stored as `"true"`/`"false"` strings in context
- For integration tests requiring runWorkflow, start a mock HTTP daemon control server in before() on a random port (TAMANDUA_CONTROL_PORT) that responds 200 to all requests — avoids needing a real daemon process
- Dynamic import of `runWorkflow` in rugpull.ts avoids circular dependency: step-ops.ts → rugpull.ts → run.ts → step-ops.ts
