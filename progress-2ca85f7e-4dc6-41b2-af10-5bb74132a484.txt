## Codebase Patterns

- Workflow YAML format: id, name, version, description, polling, agents, steps
- loadWorkflowSpec validates id, agents (non-empty array with id + workspace.baseDir), steps (with id + agent)
- Workspace files use relative paths from the workflow directory
- Agent roles: analysis (read-only), coding (read/write/exec), verification (read+exec, no write), testing, pr, scanning
- Use `tamandua` on PATH to run the CLI (installed at ~/.local/bin/tamandua), not `node bin/tamandua`
- Tests import from dist/ (build before testing)
- Generic workflow validation tests iterate over ALL directories in workflows/ - new workflows must include all workspace.files or those tests fail
- Persona-specific tests live in describes within tests/workflow-validation.test.ts; each agent gets a dedicated block
- AGENTS.md for each agent should include: responsibilities, output format, retry_feedback handling, operating modes
- Workflow structure tests check: agent count, step count and ordering, agent assignments, context passing via `{{var}}` placeholders in step inputs, no unexpected loop wiring, no merge steps
